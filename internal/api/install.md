# Install smolanalytics (self-hosted)

> OBJECTIVE: wire analytics into the app in this repo and confirm real events flow, so the
> human can just ASK their own AI questions and get exact, computed answers.

You are talking to a running self-hosted smolanalytics instance. This guide is served BY that
instance, so the host below is already correct.

## 1. Add the tracker (web app)
Put this in the HTML `<head>` (autocaptures pageviews + clicks, SPA-aware):
```html
<script src="{{HOST}}/sdk.js"></script>
<script>smolanalytics.init("{{WRITE_KEY}}", { host: "{{HOST}}" });</script>
```
- Cookieless (no consent banner): `smolanalytics.init(key, { host, anonymous: true })`
- On login: `smolanalytics.identify(userId)`
- Custom conversion events: `smolanalytics.track("signup", { plan: "pro" })`, then `track("checkout", { amount: 29 })`

Backend / mobile / any language — POST the same events:
```sh
curl -X POST {{HOST}}/v1/events -H "Authorization: Bearer {{WRITE_KEY}}" \
  -d '{"name":"signup","distinct_id":"user_123","properties":{"plan":"pro"}}'
```

## 2. Let the tools write the instrumentation for you
If you have this repo checked out, ask the running instance to read it and hand you exact edits:
- Over MCP: call `propose_instrumentation` (pass the repo path). It returns the base snippet
  location plus the exact `track()` calls to add at the real signup/checkout/activate
  call-sites (file, line, snippet). Apply them, then call `verify_instrumentation` — it
  cross-references code + live traffic and returns FIRING / WIRED / MISSING per event. Repeat
  `suggest_instrumentation_fix` until every planned event is FIRING.
- Or from the CLI: `smolanalytics instrument ./path-to-repo` prints the same edits.

## 3. Connect the human's editor to this instance over MCP
Point their Claude / Cursor / Claude Code / Windsurf / VS Code at:
- URL: `{{HOST}}/mcp` (Streamable HTTP)
- Auth header: `Authorization: Bearer {{READ_KEY}}`  (the SECRET read key — never the write key)

`smolanalytics connect` auto-merges this into every installed editor in one command. NOTE:
editors only load a new MCP server after a RESTART — tell the human to reopen their editor,
then say "finish the install" so you can verify the connection.

## 4. DONE WHEN
- The SDK snippet (or a track() call) is in the app and a test action produced a real event
  (confirm with `recent_events` over MCP, or `GET {{HOST}}/v1/events/recent`).
- The human's editor is connected to `{{HOST}}/mcp` and `overview` returns their real numbers.
- You've told the human their first real number and that they can now just ASK
  ("where do people drop off?", "is pro converting better than free?").

Real funnels/retention appear as users accumulate — a brand-new app starts with just your test
events, and that's expected. The tracking is live and correct from event one.

## The promise
Every surface — the dashboard, `GET {{HOST}}/v1/*`, and the MCP tools — returns the SAME number
for the same question, and a CI test enforces it. The answers are computed by a deterministic
engine, never generated by a model. Full tool reference: `{{HOST}}/llms.txt`.
